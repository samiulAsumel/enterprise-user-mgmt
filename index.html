Architecture

Seven Specialized Modules

Each module runs standalone or as part of the full pipeline. Every action is dry-run capable, logged to syslog, and idempotent.

create_users.sh
Bulk user creation from a CSV manifest. Handles UID assignment, home directories, shell, primary & secondary groups, GECOS fields, and initial password locking.
useradd CSV-driven idempotent
create_groups.sh
Bulk group creation and membership synchronization. Creates groups with custom GIDs, then syncs member lists — adds missing members, skips existing ones.
groupadd usermod -aG sync
sudo_config.sh
Writes /etc/sudoers.d/ fragments — validated by visudo -c before applying. Configures admin full-sudo, ops restricted commands, auditor read-only, and global security defaults.
visudo sudoers.d validated
password_policy.sh
Applies system-wide password complexity via pwquality.conf, sets global aging defaults in /etc/login.defs, and enables pam_pwhistory via authselect — all with file backups.
pwquality login.defs authselect
password_aging.sh
Per-user aging via chage from a CSV manifest. Sets max days, min days, warning period, inactive grace, and hard expiry date. Applies global defaults to all unlisted regular users.
chage per-user CSV
account_locking.sh
Configures faillock brute-force protection (deny, fail_interval, unlock_time) and processes a lock/unlock manifest with passwd -l/u and faillock --reset.
faillock passwd -l PAM
audit_report.sh — Security Audit Report Generator
Produces HTML + plain-text reports covering every regular user: group memberships, sudo access level, password status (active/locked/no-password), last change date, aging settings, expiry status, and faillock failure count. Optionally emails the HTML report. Output goes to /var/log/user-mgmt/reports/.
HTML report text report email cron-ready
Pipeline

How the Pipeline Runs

Seven steps, executed in dependency order. Any step can be run independently with --module <name>.

1
Edit Config
Set defaults in config.conf — shell, password policy, faillock settings, log paths.
2
Define Manifests
Populate data/groups.csv, users.csv, aging.csv, and locks.csv CSVs.
3
Dry Run
Execute --dry-run to preview every action with zero system changes.
4
Apply & Audit
Run the full pipeline. Review the generated HTML audit report for compliance confirmation.

Installation

    
Get Running in Minutes

    
Clone the repo, edit config.conf, populate your CSV manifests, dry-run, then apply.


    

      
      
      
    


    

      
      
# 1. Clone the repository
$ git clone https://github.com/samiulAsumel/enterprise-user-mgmt.git
$ cd enterprise-user-mgmt

# 2. Review and edit configuration
$ vi config.conf

# 3. Populate CSV manifests (sample files auto-generated on first run)
$ vi data/groups.csv
$ vi data/users.csv
$ vi data/aging.csv

# 4. Dry run — no changes applied
$ sudo bash user_mgmt_master.sh --dry-run --verbose

# 5. Install system-wide + weekly cron
$ sudo bash install.sh --install

# 6. Full run
$ sudo user-mgmt

    


    

      
      
# Run a single module without affecting others

$ sudo user-mgmt --module users      # create_users only
$ sudo user-mgmt --module groups     # create_groups only
$ sudo user-mgmt --module sudo       # sudo_config only
$ sudo user-mgmt --module policy     # password_policy only
$ sudo user-mgmt --module aging      # password_aging only
$ sudo user-mgmt --module locking    # account_locking only
$ sudo user-mgmt --module report     # audit_report only

# Or run standalone from the modules/ directory
$ sudo bash modules/account_locking.sh --dry-run
$ sudo bash modules/password_aging.sh --verbose

    


    

      
      
# /etc/cron.d/enterprise-user-mgmt (created by install.sh)
# Weekly audit report every Sunday at 01:00
SHELL=/bin/bash
PATH=/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin

0 1 * * 0  root  user-mgmt --report-only >> /var/log/user-mgmt/cron.log 2>&1

# Monthly full sync (1st of month, 02:00) — add this manually if needed
0 2 1 * *  root  user-mgmt --module aging >> /var/log/user-mgmt/cron.log 2>&1
